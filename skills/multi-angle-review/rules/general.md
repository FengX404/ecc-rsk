# 通用内容审查规则

## 审查流程

1. 主 Agent 已完成通用审查（universal_criteria.csv 抽样 3 条）
2. 本阶段启动 1 个子 Agent 审查
3. 子 Agent 独立抽样对应 CSV 中的审核标准

## 子 Agent 配置

### Agent 1: 通用质量审查

- **角色**: 通用质量审查专家，关注一致性、完整性、可维护性
- **CSV**: `references/universal_criteria.csv`
- **抽样数**: 5 条（从所有行中随机抽取）
- **Goal**: 审查内容的一致性、完整性、简洁性、可维护性
- **Context 需包含**: 被审查内容 + 抽样标准条目(JSON)

## 子 Agent 输出格式

```
## [视角名]
> 审查身份：[角色]
| 严重度 | 问题 | 位置 | 抽样标准 | 修复建议 |
|--------|------|------|----------|----------|
| 🔴严重 | ...  | 文件:行N | 标准原文 | ...      |
| 🟡建议 | ...  | 文件:行N | 标准原文 | ...      |
```

无问题则输出：「✅ 未发现相关问题。」

## 注意事项

- 本规则为 fallback，当内容无法匹配功能/质量/体验/工程四种维度时使用
- 子 Agent 必须是 leaf 角色
- 上下文语言使用中文
