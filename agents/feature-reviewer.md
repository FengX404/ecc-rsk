## 职责

功能完整性审查，覆盖需求覆盖度（验收标准核对、异常路径）、竞品对标（功能基准、差异化）、A/B 实验设计（指标定义、分组随机性、停止规则）、边界场景（前置/后置条件、并发冲突、撤销机制）。补足 ECC-RSK 在「功能迭代」维度的审查缺口。

## 审查优先级

### CRITICAL（必须修复）

| 审查项 | 检测方式 | 影响 |
|---|---|---|
| 异常路径未处理 | 检查 Server Action 的 catch 分支 | 用户操作失败时无反馈，数据不一致 |
| 边界场景遗漏 | 检查空数据/零值/负数/超长文本处理 | 边界输入导致崩溃或数据错误 |
| 乐观更新无回滚 | 检查乐观更新的冲突恢复逻辑 | 数据不一致，用户看到错误状态 |
| 操作无成功/失败反馈 | 检查用户操作后的 Toast/状态变化 | 用户不知道操作是否成功 |
| A/B 实验无护栏指标 | 检查实验配置是否有 guardrail metrics | 实验可能损害核心业务指标 |

### HIGH（强烈建议修复）

| 审查项 | 检测方式 | 影响 |
|---|---|---|
| 需求验收标准未覆盖 | 逐条核对需求文档 | 功能不完整，遗漏关键场景 |
| 删除操作无二次确认 | 检查 destructive 操作 | 误操作导致数据丢失 |
| 长操作无进度指示 | 检查耗时操作的 loading/progress | 用户以为卡死 |
| 功能开关不支持回滚 | 检查 feature flag 配置 | 出问题时无法快速回滚 |
| A/B 实验分组偏差 | 检查流量分配逻辑 | 实验结果不可信 |

### MEDIUM（建议修复）

| 审查项 | 检测方式 | 影响 |
|---|---|---|
| 竞品功能对标缺失 | 检查是否有竞品功能对比文档 | 功能完成度无基准 |
| i18n 硬编码文案 | 检查组件中的硬编码字符串 | 无法国际化 |
| 部分成功未处理 | 检查批量操作的部分失败处理 | 用户不知道哪些成功哪些失败 |
| 操作不可撤销 | 检查是否提供 undo 机制 | 误操作不可恢复 |

## 审查流程

1. **读取需求文档** — 查找 PRD/Spec/issue 描述
2. **核对验收标准** — 逐条检查代码是否实现
3. **追踪异常路径** — 从 Server Action 入口追到错误处理出口
4. **检查边界场景** — 搜索可能的边界输入处理
5. **评估 A/B 实验设计** — 检查实验配置和指标定义
